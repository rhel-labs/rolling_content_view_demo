# Rolling Content Views Demo

An interactive web-based presentation that explains Red Hat Satellite's Rolling Content Views feature. It walks the audience through how standard Content Views are published and promoted, then contrasts that with the streamlined Rolling Content View workflow using SVG diagrams.

## Audience

Mixed audience of business decision-makers and system administrators evaluating or adopting Red Hat Satellite 6.18+.

## What it covers

1. **Introduction** -- What Content Views are and why content management at scale is challenging.
2. **Standard Content View workflow** -- The publish-and-promote cycle: creating versions, promoting through lifecycle environments (Library, Dev, QA, Production), and the time and effort each cycle requires.
3. **Rolling Content View workflow** -- How rolling CVs eliminate publishing and promotion entirely, automatically reflecting the latest synced content in Library.
4. **Side-by-side comparison** -- Feature matrix, CVE response timelines, and the recommended hybrid architecture (rolling for dev/test, standard for production).
5. **Summary** -- Key takeaways, metrics, and next steps.

## Installation

No installation is required. The demo is a single self-contained HTML file.

1. Clone the repository:
   ```
   git clone <repository-url>
   cd rolling_content_view_demo
   ```
2. Open `index.html` in any modern browser:
   ```
   open index.html        # macOS
   xdg-open index.html    # Linux
   start index.html       # Windows
   ```

Alternatively, serve it with any static file server:
```
python3 -m http.server 8000
```
Then visit `http://localhost:8000` in your browser.

### Requirements

- A modern web browser (Chrome, Firefox, Safari, or Edge).
- An internet connection for loading Red Hat fonts from Google Fonts. The demo is fully functional without it but will fall back to system sans-serif fonts.

## Navigation

- **Arrow keys** or **spacebar** to step forward/backward.
- **Number keys 1-5** to jump to a section.
- **Home / End** to go to the first or last slide.
- Bottom navigation bar has clickable Previous/Next buttons and section tabs.
- URL hash (`#step-N`) updates per slide for bookmarking.

## Design

- Uses official Red Hat brand colors and typography (Red Hat Display, Red Hat Text, Red Hat Mono loaded from Google Fonts).
- All diagrams are inline SVGs generated by JavaScript helper functions.
- Single self-contained HTML file with no external dependencies beyond the font CDN.
